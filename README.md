# Portfolio Project — 100Hires Application

## Tools Installed

- **Cursor IDE** — AI-powered code editor, downloaded from cursor.com
- **Claude Code** (Cursor extension) — AI coding assistant integrated into the editor
- **Codex** (Cursor extension) — OpenAI's coding agent, integrated into the editor

## Steps Completed

1. Created a GitHub account and set up a public repository
2. Downloaded and installed Cursor IDE
3. Installed the Claude Code and Codex extensions inside Cursor
4. Installed Git on my computer
5. Cloned the repository locally using the terminal in Cursor
6. Edited this README file inside Cursor
7. Committed and pushed the changes back to GitHub

## Challenges and How I Solved Them

Coming from a marketing background with no prior coding experience, this was my first time using Git, GitHub, and a developer IDE.

The biggest challenge was the terminal. I had never used command-line tools before. When I ran the git clone command, it failed because Git was not installed on my computer. I searched for a solution, found the official Git installer at git-scm.com, installed it, restarted Cursor, and the command worked.

I also initially confused Cursor with Windsurf, another similar IDE. I noticed the interface looked different from what was described, identified the mistake, and downloaded the correct tool from cursor.com.

Finding the Extensions panel also took some trial and error. I searched YouTube and discovered the correct shortcut, then installed both Claude Code and Codex without issues.

## Reflections

This process pushed me outside my comfort zone in a productive way. I had never used Git, a terminal, or a developer IDE before starting. Every obstacle I ran into, I solved independently by searching for answers online. For someone transitioning into a role that combines marketing with modern AI tools, I believe the ability to learn fast and solve problems without hand-holding is exactly what matters most.
